# 实验设计与离线实验接口模板（密封性检测项目）

> 用途：统一记录“运行批次 - 窗口 - 工况 - 分支模型 - 结论”，避免再次回到“全局统一分类器”的试验方式。  
> 当前默认前提：近期目标是先证明实验室高信息工况可分，不把现场迁移和 MoE 作为当前阶段成败标准。

---

## 0. 使用原则

1. 所有实验先按工况角色分层，再决定是否训练模型。  
2. 低信息工况必须单独报告，不允许混入总体指标中稀释失败原因。  
3. 所有验证默认按 `leave-one-run-out` 执行，不允许同一运行批次的相邻窗口同时出现在训练集和测试集。  
4. 第一轮实验只输出三类结论：  
   - 结论 A：哪些工况应直接排除  
   - 结论 B：哪些工况存在稳定可分性  
   - 结论 C：状态转移时分数能否在变化区域附近持续抬升

## 2. 离线实验输入接口规范

## 2.1 Run Manifest

> 一个运行批次对应一条记录。默认存为 CSV 或 Parquet，主键为 `run_id`。

| 字段 | 含义 | 示例 |
|---|---|---|
| `run_id` | 运行批次唯一标识 | `lab_20260324_case03` |
| `file_path` | 原始数据文件路径 | `data/lab/case03.csv` |
| `seal_state_global` | 该运行的全局状态标签 | `sealed` / `unsealed` / `mixed` |
| `has_transition` | 是否包含状态转移 | `true` |
| `transition_ts` | 已知转移时刻，未知则留空 | `1711262400000` |
| `condition_family_manual` | 人工归属的工况族 | `ext_high_hum_no_heat` |
| `heat_state_manual` | 热源状态 | `off` / `on` / `mixed` |
| `ext_humidity_regime_manual` | 外部湿度背景 | `stable` / `high` / `fluctuating` |
| `notes` | 备注 | `mid-run drilling at 14:30` |

## 2.2 Window Table

> 每个滑动窗口一条记录。默认由离线脚本生成，主键建议为 `run_id + window_id`。

| 字段 | 含义 | 示例 |
|---|---|---|
| `run_id` | 对应的运行批次 | `lab_20260324_case03` |
| `window_id` | 窗口唯一标识 | `lab_20260324_case03_w_00128` |
| `start_ts` | 窗口起始时间 | `1711261800000` |
| `end_ts` | 窗口结束时间 | `1711262400000` |
| `W` | 窗口大小 | `120` |
| `S` | 滑动步长 | `5` |
| `segment_id` | 所属阶段或变化点分段编号 | `seg_03` |
| `candidate_condition` | 自动或人工给出的候选工况 | `cooling_after_heat_stop` |
| `info_score` | 信息量评分 | `0.73` |
| `label_coarse` | 粗粒度标签 | `low_info` / `high_value` / `transition_neighbor` |
| `label_confidence` | 标签置信度 | `manual_high` / `manual_low` / `auto_rule` |

## 2.3 推荐标签约定

- `seal_state_global`：`sealed`、`unsealed`、`mixed`
- `condition_family_manual`：`stable_no_heat`、`stable_with_heat`、`ext_high_hum_no_heat`、`cooling_after_heat_stop`、`internal_moisture`、`complex_coupled`
- `label_coarse`：`exclude_low_info`、`candidate_high_info`、`transition_neighbor`、`holdout_complex`

## 12. 组会汇报最小话术（1分钟）

1. 当前难点不是模型不够大，而是多数窗口本身不可分。  
2. 解决路径是先做低信息工况排除，再在高信息工况和状态转移段内比较三类分支。  
3. 所有实验按运行批次做 `leave-one-run-out`，避免相邻窗口泄漏。  
4. 只有当至少两个分支在同一工况族里表现出互补性，才进入轻量融合，更复杂的 MoE 放到后面。
