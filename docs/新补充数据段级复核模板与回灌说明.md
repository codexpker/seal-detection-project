# 新补充数据段级复核模板与回灌说明

当前新补充数据已经形成一条独立、可回灌的段级复核链：

`segment_static_support_v2 -> segment_review_queue_v2 -> review_labels_template -> segment_feedback_loop_v1`

在当前已经切到更保守的 `v3 guarded` 之后，默认应改用：

`segment_static_support_v3 -> segment_review_queue_v3 -> review_labels_template_v2 -> segment_feedback_loop_v2`

如果要把“生成模板 / 自动种子 / 回灌 / 人工复核 agenda”一次跑齐，现在默认可直接使用：

`new_data_review_workflow_v1`

## 1. 当前入口文件

- 段级复核队列：
  - `reports/new_data_segment_static_support_v3_run1/segment_review_queue_v3.csv`
- 段级复核模板：
  - `reports/new_data_segment_review_label_template_v2_run1/segment_review_labels_template_v2.csv`
- 段级回灌输出目录：
  - `reports/new_data_segment_feedback_loop_v2_run1/`

## 2. 需要填写的列

回灌脚本真正读取的只有 4 列：

- `segment_id`
- `review_label`
- `reviewer`
- `review_note`

其他列都是辅助上下文，保留即可。

## 3. 推荐标签

- `transition_positive`
  - 用于主战场 `transition_primary_support`
- `positive_reference`
  - 用于确认该段可进入静态正参考池
- `negative_reference`
  - 用于确认该段可进入静态负参考池
- `breathing_watch`
  - 用于确认该段属于密封呼吸/材料呼吸难例
- `confound`
  - 用于确认该段属于 `heat-off` 或类似混淆段
- `uncertain`
  - 当前无法稳定判断，先保留待复核

## 4. 当前建议优先级

按当前 `segment_review_queue_v3`，建议先确认：

1. 三个 `transition_primary_support`
2. `20260303-181049` 这一条 `static_review_weak_positive_guarded`
3. `20260306-160246` 这一条 `static_watch_breathing_hardguard`
4. `20260321-170450` 这一条 `static_watch_confound_hardguard`

## 9. 当前阶段的正确理解

这一步不是训练新模型，而是把你的人工判断沉淀成：

- 可扩容的静态正/负参考池
- 可复用的 `breathing` 难例池
- 可复用的 `confound` 混淆池
- 下一轮更合理的待复核排序

因此它服务的是当前第二阶段“把闭环跑稳”的目标，而不是进入第三阶段模型融合。

## 12. 当前默认下一步

2. 按 `reports/new_data_review_workflow_v1_run1/review_agenda_v1.md` 完成真实人工复核

5. 只有 guarded 段连续稳定后，才考虑调 `guard_score_thresh / weak_support_thresh`

- 不重启 whole-run `GRU / XGBoost`
- 不把 `181049` 自动并入 `positive_reference`
- 不把 `transition_secondary_control` 提前并入主战场正池

## 13. 复核后自动回灌

如果你已经在 `segment_review_labels_working_v2.csv` 里完成或部分完成人工复核，现在默认可以直接运行：

```bash
python src/scripts/new_data_review_finalize_v1.py \
  --readme-xlsx /Users/xpker/Downloads/data_readme.xlsx \
  --segment-manifest-csv reports/new_data_segment_pipeline_v1_run1/segment_pipeline_manifest.csv \
  --segment-support-csv reports/new_data_segment_static_support_v3_run1/segment_support_output_v3.csv \
  --review-queue-csv reports/new_data_segment_static_support_v3_run1/segment_review_queue_v3.csv \
  --working-labels-csv reports/new_data_review_workflow_v1_run1/segment_review_labels_working_v2.csv \
  --output-dir reports/new_data_review_finalize_v1_run1
```

它会自动做三件事：

1. 读取当前人工编辑后的 `segment_review_labels_working_v2.csv`
2. 重跑 `new_data_segment_feedback_loop_v2.py`
3. 刷新最新的 `tuning_plan_v1` 和收口报告

核心输出：

- `reports/new_data_review_finalize_v1_run1/new_data_review_finalize_v1.md`
- `reports/new_data_review_finalize_v1_run1/segment_review_labels_snapshot_v1.csv`
- `reports/new_data_review_finalize_v1_run1/feedback_v2/pending_segment_review_reranked_v2.csv`
- `reports/new_data_review_finalize_v1_run1/tuning_plan_v1/new_data_model_tuning_plan_v1.md`

如果要从后端直接触发，而不是手工进命令行，现在可以调用：

```bash
curl -X POST http://localhost:8000/api/review/new-data/finalize \
  -H 'Content-Type: application/json' \
  -d '{
    "readme_xlsx": "/Users/xpker/Downloads/data_readme.xlsx",
    "segment_manifest_csv": "reports/new_data_segment_pipeline_v1_run1/segment_pipeline_manifest.csv",
    "segment_support_csv": "reports/new_data_segment_static_support_v3_run1/segment_support_output_v3.csv",
    "review_queue_csv": "reports/new_data_segment_static_support_v3_run1/segment_review_queue_v3.csv",
    "working_labels_csv": "reports/new_data_review_workflow_v1_run1/segment_review_labels_working_v2.csv",
    "output_dir": "reports/new_data_review_finalize_v1_run1"
  }'
```

后端会直接返回：

- 最新 `summary`
- 最新 `outputs`
- 当前执行时间戳 `requested_at`
