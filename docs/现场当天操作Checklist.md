# 现场当天操作 Checklist

## 1. 当天目标

只完成当前系统最能支撑演示的主线：

`电机1 / 高湿 / 密封 -> 不密封切换`

不额外扩展到：

- 电机2 加水工况
- 现场长期监测
- 统一 seal / unseal 分类验证

---

## 2. 开始前确认

开始前只确认这几件事：

1. 时间记录设备正常
2. 温湿度采集正常
3. 外部高湿条件可维持
4. 切换动作可以精确记录绝对时间
5. 数据文件命名清楚，能区分 `seal` / `unseal` / `transition`

---

## 3. 当天执行顺序

## Step 1：密封段

- 保持密封状态
- 进入高湿环境
- 先完成密封段采集
- 优先保留：
  - 夜间高湿静态段
  - 白天高湿运行段
  - 白天高湿停机静态段

当前目的不是“所有段都判”，而是为后续切换提供前状态基线。

## Step 2：记录切换时刻

- 到达切换点时，明确记录绝对时间
- 切换动作为：
  - 打开接线盒
  - 或拆除密封胶条
- 必须把该时间记成唯一可信时间点

这是当前最关键的操作，不能模糊。

## Step 3：不密封段

- 切换后继续保持同类高湿背景
- 继续采集：
  - 夜间高湿静态段
  - 白天高湿运行段
  - 白天高湿停机静态段

当前最重要的是保证切换前后工况尽量一致，减少额外变量。

---

## 4. 当天不做什么

当天不要主动做下面这些事：

1. 不临时改实验目标
2. 不补做加水工况
3. 不为了“看起来更丰富”增加很多不稳定工况
4. 不现场讨论统一分类准确率
5. 不把热相关段当成主演示证据

---

## 5. 采集结束后固定动作

采集结束后固定只做这几步：

1. 整理文件名
2. 记录切换绝对时间
3. 确认哪些文件属于：
   - `seal`
   - `transition`
   - `unseal`
4. 跑演示入口
5. 导出总览和重点卡片

固定命令：

```bash
MPLCONFIGDIR=/tmp/mplconfig python src/scripts/lab_demo_entry_v1.py --output-dir reports/lab_demo_entry_v1_run1
MPLCONFIGDIR=/tmp/mplconfig python src/scripts/lab_interference_regression_entry_v1.py --output-dir reports/lab_interference_regression_entry_v1_run1
```

---

## 6. 当天验收标准

当天只按下面标准判断是否成功：

1. 是否有明确切换时间
2. 是否采到切换前后连续数据
3. 是否能输出 `transition_boost_alert`
4. 是否能展示至少一个静态辅助样例
5. 是否能展示至少一个 `watch / abstain` 保守样例

只要这 5 条满足，现场演示就够用。

---

## 7. 当天汇报口径

当天只能这样讲：

- 当前最强能力是状态切换检测
- 静态场景提供辅助风险
- 难例和干扰场景主动保守

不要这样讲：

- 已经稳定解决全部密封性检测
- 所有工况都可分类
- 现场长期应用已经闭环

---

## 8. 最后一条

如果当天时间不够，优先保：

1. 切换前高湿静态段
2. 明确切换时刻
3. 切换后高湿静态段

其他内容都可以让位给这三项。
