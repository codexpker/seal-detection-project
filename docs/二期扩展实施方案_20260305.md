# 二期扩展实施方案（2026-03-05）

> 面向“实验室已标注数据 + 现场复杂流式数据”的密封性异常检测升级方案。
> 本文档聚焦可落地实施，作为二期执行主文档。

---

## 1. 二期目标与范围

### 1.1 目标

在不推翻一期链路（MQTT 入库 → 自动检测 → SSE 推送 → 故障归档）的前提下，新增一套更鲁棒的“现场异常检测”能力，重点解决：

1. 实验室数据与现场数据分布差异大导致的泛化问题。
2. 现场采样不均匀、传感器偏移、外界干扰（如胶泥）导致的误报漏报。
3. 仅依赖单模型硬分类难以稳定运行的问题。

### 1.2 范围

- **包含**：特征重构、两阶段检测、在线漂移监测、事件级告警策略、评估与回放验证。
- **不包含**：权限体系、多租户治理、跨项目统一告警平台。

---

## 2. 现状与问题复盘

一期已完成：

- MQTT 入库（`reader.py`）
- 自动触发检测（后端排队）
- 检测结果落库（`detection_result_log`）
- SSE 实时展示（首页/诊断页）
- 故障专档落库（`fault_archive`）

现阶段问题：

1. GRU、LightGBM 在实验室离线效果可接受，但现场稳定性不足。
2. 等间隔训练范式与现场不规则采样冲突。
3. 绝对值特征易受传感器偏移影响，跨设备泛化较弱。
4. 业务真正关注的是“事件告警质量”，而非点级二分类准确率。

---

## 3. 技术路线总览（推荐执行）

二期采用“**特征重构 + 两阶段检测 + 在线校准**”路线：

1. **特征层重构**：引入物理先验与相对量特征，降低绝对值依赖。
2. **阶段一（点级）**：生成连续异常分数（anomaly score），优先无监督/弱监督。
3. **阶段二（事件级）**：窗口聚合 + 规则/轻量模型，输出最终告警。
4. **漂移监测与阈值校准**：发现分布变化后进行阈值或模型参数自适应。

---

## 4. 数据输入输出定义（执行口径）

### 4.1 输入字段（最小必需）

- `dev_num`：设备号
- `ts`：时间戳（ms）
- `temp_in`：盒内温度
- `temp_out`：盒外温度
- `hum_in`：盒内湿度
- `hum_out`：盒外湿度
- `dt`：与上一样本间隔（秒）

可选补充：

- 环境状态标签（雨天/昼夜/工况）
- 传感器健康标志（自检状态、失联次数）

### 4.2 阶段一输出（点级）

- `score_raw`：原始异常分数（0~1 或无界后归一化）
- `score_smooth`：平滑分数（EWMA）
- `drift_flag`：是否触发漂移检测
- `feature_snapshot`：本点关键特征快照（用于诊断）

### 4.3 阶段二输出（事件级）

- `event_id`：事件ID（设备+起始时间）
- `event_start_ts` / `event_end_ts`
- `event_peak_score`
- `event_duration_sec`
- `event_level`：`info|warn|critical`
- `alarm_decision`：`0|1`
- `decision_reason`：规则触发说明（可解释性）

## 5. 特征工程方案（重点）

### 5.1 相对量与物理先验特征

1. 温差：`delta_t = temp_in - temp_out`
2. 湿差：`delta_h = hum_in - hum_out`
3. 归一湿差：`delta_h_norm = delta_h / (abs(hum_out) + eps)`
4. 温湿耦合项：`delta_t * delta_h`

### 5.2 动态特征

1. 一阶斜率：`slope_x = (x_t - x_{t-1}) / dt`
2. 二阶变化：`acc_x = (slope_t - slope_{t-1}) / dt`
3. 短窗波动率：`std_5m / std_30m`
4. 稳态偏移：最近窗口均值与历史基线差

### 5.3 时滞与恢复特征

1. 内外湿度互相关峰值与时滞
2. 扰动后恢复时间常数（达到稳态阈值所需时长）
3. 连续静止段长度（识别传感器卡死或异常平坦）

### 5.4 质量特征

1. 采样间隔异常率（`dt` 超阈值比例）
2. 缺失/插值比例
3. 传感器可用性分数

## 6. 模型与服务接口设计

## 6.1 阶段一：点级异常打分

推荐先上两条并行基线：

- **统计基线**：Robust Z-Score + EWMA + CUSUM
- **在线模型**：River（如 HalfSpaceTrees / 在线 Isolation 思路）

输出统一为 `score_raw`，再经平滑得到 `score_smooth`。

### 6.2 阶段二：事件判定

1. 以 `score_smooth` 超阈触发候选事件开始。
2. 低于恢复阈值并持续一段时间判定结束（迟滞机制）。
3. 聚合事件特征（峰值、持续时间、恢复速度、上下文条件）。
4. 规则优先 + 轻量模型（LogReg/LightGBM）给出 `alarm_decision`。

### 6.3 后端接口草案

- `POST /api/internal/anomaly/v2/score/{dev_num}/{device_timestamp}`
  - 输入：最新窗口数据
  - 输出：点级分数与漂移标志

- `POST /api/internal/anomaly/v2/event/{dev_num}/{device_timestamp}`
  - 输入：点级分数流
  - 输出：事件级告警决策

- `GET /api/runtime/anomaly-v2/metrics`
  - 输出：二期运行指标（误报、漏报估计、漂移触发次数、平均延迟）

## 7. 阈值与告警策略（默认建议值）

> 以下为初始默认值，需在现场按误报率目标校准。

### 7.1 点级阈值

- `warn_threshold = 0.65`
- `critical_threshold = 0.80`
- EWMA 平滑系数：`alpha = 0.25`

### 7.2 事件迟滞参数

- 连续高于 `warn_threshold` 达 `N_start=3` 个点触发事件
- 连续低于 `recover_threshold=0.45` 达 `N_end=5` 个点结束事件
- 最短事件时长：`min_event_duration_sec = 180`

### 7.3 抑制与合并

- 同设备事件冷却时间：`cooldown_sec = 600`
- 高峰期间同类告警合并（避免告警风暴）

## 8. 评估体系（由点级改为事件级）

### 8.1 核心指标

1. 事件级 Precision / Recall / F1
2. 平均告警提前量（越高越好）
3. 误报率（次/设备/天）
4. 漏报率（按人工确认事件）

### 8.2 辅助指标

1. 点级 PR-AUC
2. 分数稳定性（方差/抖动）
3. 模型延迟（ms）与吞吐（QPS）

### 8.3 回放验证

沿用既有回放能力，新增二期策略对照：

- 同一历史区间同时跑“一期策略 vs 二期策略”
- 自动输出差异报告（新增命中、减少误报、延迟变化）

## 9. 2~4 周执行计划

### 第1周：特征与评估重建

- 完成特征字典与计算管道
- 建立事件级评估脚本
- 选取 3~5 台设备做离线回放基线

### 第2周：上线统计基线

- 接入 Robust Z + EWMA + CUSUM
- 打通点级分数入库与可视化
- 小流量灰度验证误报水平

### 第3周：接入在线模型与漂移检测

- 引入 River 在线异常模型
- 增加 ADWIN/等价机制做漂移检测
- 触发阈值动态校准（受控）

### 第4周：迁移与闭环

- 利用实验室标签做阈值初始化与特征筛选
- 人工复核 Top-K 高分事件形成现场金标准
- 固化版本并输出二期验收报告

## 10. 与现有系统的改造原则

1. **不破坏一期接口**：二期以旁路方式接入，支持随时回退。
2. **结果可解释**：每次告警必须可追溯到特征与规则。
3. **渐进上线**：先影子运行，再灰度，再全量。
4. **统一归档**：点级分数与事件级判定都写库，支持审计与复盘。

## 11. 开源与文献参考（用于二期选型与评估）

### 11.1 开源项目

1. **River（online-ml/river）**
   - 用途：在线学习、流式异常检测、概念漂移检测。
   - 适配点：现场流式数据、采样不稳定、需持续更新的场景。

2. **Merlion（salesforce/Merlion）**
   - 用途：时序异常检测与预测统一框架，便于快速对比多算法。
   - 适配点：离线评估与策略选型阶段。

3. **TSB-AD（TheDatumOrg/TSB-AD）**
   - 用途：时间序列异常检测基准与评估工具集。
   - 适配点：评估方法（尤其 PR 视角与不同算法公平对比）。

### 11.2 文献与方法（建议优先级）

1. **Liu & Paparrizos, NeurIPS 2024, TSB-AD**
   - 结论启发：复杂模型并不总优于简单稳健方法，评估体系同样关键。

2. **TranAD, VLDB 2022**
   - 方向：多变量时序异常检测的 Transformer 路线，可作深度模型对照组。

3. **DAGMM, ICLR 2018**
   - 方向：经典无监督异常检测基线，可用于离线对比与特征有效性验证。

> 备注：二期优先“稳健可运维”而非“模型复杂度”。先把统计+在线方案跑稳，再逐步引入深度模型。

## 12. 风险与应对

1. **风险：误报偏高影响运维信任**
   - 应对：迟滞 + 冷却 + 事件合并 + 分级告警。

2. **风险：现场标签稀缺，难做监督训练**
   - 应对：无监督打分 + 主动学习复核 + 周期校准。

3. **风险：不同设备差异大**
   - 应对：设备分组阈值、设备级基线、分层评估。

4. **风险：漂移后性能突降**
   - 应对：漂移检测触发降级策略（回退统计基线）与阈值重估。
