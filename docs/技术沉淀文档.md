# Seal Detection System 技术沉淀与实施状态（统一文档）

> 项目：电机接线盒密封性异常检测系统  
> 技术栈：FastAPI + Vue3 + MySQL + ECharts  
> 最后更新：2026-03-08

---

## 1. 当前系统闭环

当前主流程已形成可运行闭环：

1. MQTT 消息接入（`reader.py`）
2. 原始数据入库（`device_monitoring_data`）
3. 自动触发检测（`/api/internal/process/{dev_num}/{device_timestamp}?queued=1`）
4. 后端队列合并与节流处理（`backend_app.py`）
5. 检测结果落库（`detection_result_log`）
6. 异常事件/故障档案落库（`anomaly_event` / `fault_archive`）
7. 首页与诊断页 SSE 推送

---

## 2. 近期关键改动（2026-03）

### 2.1 首页与设备轮播

- 设备轮播从“每设备一条最新值”调整为“按检测事件流展示”
- 新增 `ticker_event` SSE 事件，轮播可增量接收最新检测
- `/api/home/device-ticker` 改为按 `device_timestamp DESC` 返回最近事件

### 2.2 首页大图调度与高频稳定性

- 首页大图仍保持最短展示时长策略（`HOME_MIN_DISPLAY_SECONDS`）
- 队列节流命中时，不再直接丢弃最新事件，而是回写待处理队列，避免“最新一条未处理”

### 2.3 设备查询与首页联动

- 设备查询接口 `GET /api/device/detail/{dev_num}` 支持 `end_ts` 锚点参数
- 首页轮播点击设备号可跳转设备查询页并自动执行查询
- 跳转后可对齐首页“实时片段”查询语义

### 2.4 MQTT 去重策略（最新）

`reader.py` 已实现“连续重复压缩 + 5分钟心跳保留”：

- 业务值变化：立即保留并触发检测
- 业务值连续不变：中间重复压缩，仅每 5 分钟保留一条样本
- 目标：减少无效重复写库，同时可观测“长期保持不变”状态

## 3. 数据模型（现状）

### 3.1 核心表

1. `device_monitoring_data`：原始采样
2. `detection_result_log`：检测流水
3. `anomaly_event`：设备+小时异常去重事件
4. `fault_archive`：故障档案
5. `device_model_preference`：设备级模型偏好

### 3.2 关键规则

- 检测窗口：N+T（条数+时间）
- 异常标注：同设备同小时最多一条展示标注
- 高频触发：同设备合并队列 + 最小处理间隔
