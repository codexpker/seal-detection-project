# 密封性异常检测复合模型架构图（论文风格）

> 说明：该图已按“实验室可分性优先”的路线重构。  
> 核心思想不再是全局统一分类，而是  
> **低信息工况排除 + 高信息工况优先 + 多分支基线 + 事件级融合判定**。

## Figure 1. Lab-first Overall Framework

```mermaid
flowchart LR
    A[Raw Multivariate Time Series\nTin Tout Hin Hout ts] --> B[Preprocessing\nresample denoise missing handling]
    B --> C[Condition Segmentation\nchange-point stage split]
    C --> D{Condition Gate\nlow-info or high-info?}

    D -- Low-info --> X[Exclude or Mark Only\nnot for anomaly training]
    D -- High-info --> E[Candidate Window Selector\nW S info-score]
    D -- Complex-Coupled --> Y[Holdout Pool\nfor later deep validation]

    E --> F1[Branch A\nRules + XGBoost/LightGBM]
    E --> F2[Branch B\nNormality Modeling\nGRU/TCN Residual]
    E --> F3[Branch C\nSimilarity Retrieval\nHealthy Window Library]

    F1 --> G[Lightweight Fusion\nLR or LightGBM first]
    F2 --> G
    F3 --> G

    G --> H[Point-level Risk Score\nscore_raw score_smooth]
    H --> I[Event State Machine\nstart end hysteresis cooldown]
    I --> J[Event-level Output\nevent_id peak duration level]

    J --> K[Human Review Labeling\ntrue false uncertain]
    K --> L[Feedback Loop\nthreshold tuning recalibration]
    L --> G
```

## Figure 2. Transition-aware Event Logic

```mermaid
flowchart TD
    S0[Input score_smooth_t] --> S1{in_event?}

    S1 -- No --> S2{score >= warn_threshold}
    S2 -- Yes --> S3[above_count += 1]
    S2 -- No --> S4[above_count = 0]
    S3 --> S5{above_count >= N_start\nand cooldown_ok}
    S5 -- Yes --> S6[Start Event\nrecord start_ts peak]
    S5 -- No --> S0

    S1 -- Yes --> S7[peak = max peak score]
    S7 --> S8{score < recover_threshold}
    S8 -- Yes --> S9[below_count += 1]
    S8 -- No --> S10[below_count = 0]
    S9 --> S11{below_count >= N_end}
    S10 --> S11
    S11 -- No --> S12[Ongoing Snapshot\ncheck transition-neighbor persistence]
    S11 -- Yes --> S13[Close Event\nwrite final event]
    S12 --> S0
    S13 --> S0
```

## 图注建议（可直接放论文）

- **Figure 1** 展示了实验室优先的总体框架：先做工况 gate，再对高信息窗口执行多分支建模，低信息工况不进入异常训练。
- **Figure 2** 给出事件级迟滞判定逻辑，并强调状态转移邻域要看持续抬升，而不是只看孤立峰值。
- **Figure 3** 明确三阶段路线：先证明可分性，再做受控三分支比较，最后才允许进入融合层和更复杂的 MoE。

---

## 路线约束（写给后续实现）

1. 低信息工况不是“效果差的数据”，而是当前阶段应显式排除的训练对象。  
2. `GRU` 预测残差分支保留，但只作为三分支之一，不再承担统一答案角色。  
3. 融合层默认从 `Logistic Regression` 或 `LightGBM` 开始。  
4. `MoE / Stacking / Boosting` 只有在至少两个分支存在互补性证据后才允许进入。
