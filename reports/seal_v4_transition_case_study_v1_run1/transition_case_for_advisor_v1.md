# seal_v4 在线模型调优简报（导师版）

## 这次做了什么

- 没有重训整套 `GRU / XGBoost`
- 没有改前后端接口
- 只在当前在线 `seal_v4` 里补了一个小步规则增强
- 目标是把已经在线下验证有效的 `露点 / 滞后 / 耦合 / 持续性` 特征真正用于当前系统

## 为什么要补这一步

- `online.2` 已经能把 `181049` 这类 guarded positive 更稳地压回 `watch`
- 但主战场里还有一条 `20260323-213435` 属于“慢爬升 transition”：
  - 分数其实已经上来了
  - 但旧规则要求 `delta_in_h >= 0.6`
  - 该样本长期停在 `0.55 ~ 0.59`
  - 所以被漏掉

## 这次怎么补

- 保留原来的快跳变判定不动
- 只新增一个更保守的慢爬升触发条件：
  - `delta_in_h >= 0.55`
  - `delta_ah_in >= 0.08`
  - `score >= 0.28`
  - 连续 `3` 个窗口满足才触发

## 结果

- 主战场 `transition_primary_mainfield` 从 `2/3` 补到 `3/3`
- `20260323-213435` 从 `0` 次命中补到 `11` 次命中
- `transition_secondary_control` 仍然是 `0 -> 0`
- `181049` 这类 guarded positive 仍然会被压回 `watch`
- 说明这次优化同时保住了两件事：
  - 尾窗更保守
  - 主战场 transition 更完整

## 对当前系统的意义

- 这次优化已经进入当前真实调用的在线模型
- 会同时作用于：
  - `xlsx` 上传回放
  - MQTT 实时检测
- 不需要额外改业务逻辑

## 当前建议

- 当前最值得继续推进的不是重训大模型
- 而是继续沿着：
  - `transition 主导`
  - `static guarded support`
  - `watch / confound challenge`
 这三条线做针对性加固
