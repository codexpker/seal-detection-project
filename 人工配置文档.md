# 人工配置文档（MVP 后端）

本文档只覆盖项目运行前需要人工确认/配置的内容。

## 1. 环境要求

- Python 3.10+
- MySQL 8.0+
- 已有业务库：`bst`

## 2. 数据库准备

在 MySQL 中执行迁移脚本：

- 文件：`migration_mvp_backend.sql`
- 作用：
  1. 为 `device_monitoring_data` 增加查询索引
  2. 创建 `detection_result_log`（检测流水）
  3. 创建 `anomaly_event`（异常事件，每小时去重）

> 如果 `device_monitoring_data` 上已存在同名索引 `idx_dev_ts`，请先确认后再执行脚本，避免重复创建报错。

## 3. 后端环境变量

`backend_app.py` 使用以下环境变量（未配置时会使用默认值）：

- `MYSQL_HOST`（默认 `localhost`）
- `MYSQL_USER`（默认 `root`）
- `MYSQL_PASSWORD`（默认 `root`）
- `MYSQL_DATABASE`（默认 `bst`）
- `MODEL_SERVICE_URL`（默认 `http://localhost:9000`）
- `DEFAULT_MODEL_NAME`（默认 `auto`，可选 `auto/xgboost/gru`）
- `WINDOW_N`（默认 `120`）
- `WINDOW_T_MINUTES`（默认 `10`）
- `HOME_MIN_DISPLAY_SECONDS`（默认 `60`）
- `POLLER_ENABLED`（默认 `true`）
- `POLLER_INTERVAL_MS`（默认 `1000`）

## 4. 模型服务联通要求

后端会调用：

- `POST {MODEL_SERVICE_URL}/model/predict`

返回 JSON 最少需要包含字段：

- `is_anomaly`（bool）
- `score`（float）
- `threshold`（float）
- `model_name`（string）
- `model_version`（string/null）
- `latency_ms`（int）

## 5. 数据接入说明（reader）

如果你要使用 MQTT 实时入库流程，需要单独运行 `reader.py`。

`reader.py` 当前文件内写死了 MQTT 和 MySQL 配置（不是环境变量模式），上线前建议手工确认：

- MQTT 地址、端口、用户名、密码、topic
- MySQL 库连接配置

## 6. MVP 功能边界确认

本次交付只覆盖 MVP 必需能力：

- 实时检测编排（轮询未处理采样点）
- 检测流水落库
- 异常事件按设备+小时去重
- 主页/设备/诊断流/模型管理接口

不包含：

- 权限鉴权体系
- 复杂告警编排
- 自动训练与模型发布平台
